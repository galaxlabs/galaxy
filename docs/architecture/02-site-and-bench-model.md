# Site and Bench Model

## Concepts

**Bench** — A workspace directory containing a Galaxy framework installation, one or more site configs, app code, and logs. Created by `galaxy bench init`.

**Site** — A named domain (e.g. `default.local`) with its own PostgreSQL database, config file, and installed apps. Each site is fully independent.

## Directory Layout

```
galaxy/
├── sites/
│   ├── platform.db                # SQLite registry of all managed sites
│   ├── default.local/
│   │   ├── site_config.json       # DB credentials, apps, settings
│   │   └── backups/               # pg_dump .dump files
│   └── mysite.local/
│       ├── site_config.json
│       └── backups/
├── site_config.json               # (reserved for future common config)
└── common_site_config.json        # Shared settings (security defaults, dev mode)
```

## Platform DB (SQLite)

File: `sites/platform.db`

Schema:

```sql
CREATE TABLE sites (
    name TEXT PRIMARY KEY,
    status TEXT DEFAULT 'active',
    db_host TEXT DEFAULT '127.0.0.1',
    db_port INTEGER DEFAULT 5432,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

Used by: `internal/bench/platform_db.py`

## Site Config (JSON)

File: `sites/<name>/site_config.json`

```json
{
    "site_name": "default.local",
    "db_name": "galaxy_default",
    "db_host": "127.0.0.1",
    "db_port": 5432,
    "db_user": "galaxy_default_user",
    "db_password": "<auto-generated>",
    "apps": ["core"],
    "created_at": "..."
}
```

Generated by `create_site_config()` in `internal/bench/site_manager.py`.

## Site Lifecycle

```
1. bench init          → Create sites/ dir and platform.db
2. bench create-site   → Generate config JSON with random PG credentials
   (User manually creates PostgreSQL database + user)
3. galaxy install      → Create core tables, seed data (runs on existing site)
4. bench status        → Check site health
5. bench install-app   → Register app in config + tabInstalled App table
6. bench backup        → pg_dump custom format
7. bench restore       → pg_restore --clean --if-exists
8. bench delete        → Remove config + platform record (DB untouched)
9. bench migration-status → Show applied/pending migrations per DocType
```

## Database Per Site

Each site gets its own PostgreSQL database. The framework connects using credentials from `site_config.json`. The SQLAlchemy engine is created by `get_engine(site)` in `internal/db/connection.py`.

Connection string format:
```
postgresql+psycopg://{user}:{password}@{host}:{port}/{db_name}
```

## Config Loading

- `load_common_config()` — reads `common_site_config.json` for security defaults
- `load_site_config()` — reads current site's config (site name from env or default)
- `get_security_settings()` — reads CSRF, rate limit, script permissions from common config
