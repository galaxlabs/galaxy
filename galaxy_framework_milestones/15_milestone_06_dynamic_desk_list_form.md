# Milestone 6 — Dynamic Desk List/Form

## Goal

Render list and form views automatically from DocType metadata.

## Build

- generic list view
- generic form view
- field renderer
- Data input
- Select input
- Check input
- Date/Datetime input
- Link input basic
- save document
- edit document
- status badge/basic indicators

## Rule

UI must read DocType and DocField metadata, not hardcode Customer/Supplier fields.

## Acceptance Criteria

- Customer list opens
- Customer form opens
- create record from Desk
- edit record from Desk
- API and UI use same CRUD layer
- tests pass
